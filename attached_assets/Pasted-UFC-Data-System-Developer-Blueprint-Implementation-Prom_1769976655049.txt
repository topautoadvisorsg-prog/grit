UFC Data System — Developer Blueprint & Implementation Prompt
1) Objective

We are building a UFC fight data system.
The goal is to extract, structure, store, and display UFC fight statistics exactly as they appear in official UFC-style data.

This system must support:

Fight metadata

Fighter profiles

Fight results

Totals stats

Significant strike breakdowns

Per-round statistics

README is the system roadmap.
Every field listed here must exist in the database + API + UI.

2) Core Entities (Data Model)
A) Event

Fields:

event_id

event_name
→ Example: UFC 324: Gaethje vs. Pimblett

event_date (if available later)

location (optional future field)

B) Fight

Fields:

fight_id

event_id (FK)

fight_type
→ Example: UFC Interim Lightweight Title Bout

weight_class
→ Example: Lightweight

title_fight (boolean)

rounds_scheduled
→ Example: 5

round_end
→ Example: 5

time_end
→ Example: 5:00

time_format
→ Example: 5 Rnd (5-5-5-5-5)

referee
→ Example: Marc Goddard

method
→ Example: Decision - Unanimous

C) Fighter

Fields:

fighter_id

name
→ Example: Justin Gaethje

nickname
→ Example: "The Highlight"

corner
→ Red / Blue (optional)

result
→ W / L

D) Judges Scorecards

Fields:

scorecard_id

fight_id

judge_name
→ Example: Mike Bell

fighter1_score
→ Example: 47

fighter2_score
→ Example: 48

Example records:

Mike Bell → 47 - 48

Derek Cleary → 46 - 49

Sal D'amato → 46 - 49
